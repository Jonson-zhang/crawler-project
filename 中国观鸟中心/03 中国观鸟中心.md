## 二、加密参数观察

打开页面后，点击“翻页”，会调用 `search` 接口，观察 `search`，

- **请求头**： 请求方法是 post，请求头中有参数 `sign` 和 `timestamp` 两个参数。其中 timestamp 是时间戳，每次请求时，`sign` 和 `timestamp` 两个参数都会变化

- **content-type**： application/x-www-form-urlencoded; charset=UTF-8 

- **载荷**：共 172 位长的字符串，每次请求时，数据都不一样，需要逆向

![[Pasted image 20260614010713.png]]

- **响应数据是加密，保存在响应体中的 data 字段中**，如下所示：

![[Pasted image 20260614004045.png]]

## 三、逆向

根据上面的分析，需要处理请求头中的 sign、载荷中字符串、以及响应数据的解密

- 请求头中的 sign，第一反应是采用全局搜索，
- 响应数据的解密采用 hook json


### 1、请求头关键字搜索

在浏览器控制台执行 `Ctrl+Shif+F`，在“搜索”中，输入： `\bsign\b file:*.js`，得到搜索结果：

![[Pasted image 20260614005246.png]]

观察前三项，都是给原型设置方法的代码，不是我们需要的 ，对sign 参数的赋值。只有第四项符合要求。点击进入代码：

![[Pasted image 20260614005452.png]]

设置断点，单步执行，发现：

- timestamp 是一个时间戳 

- requestId 是一个 32 为的 uuid

- data=JSON.stringify(sort_ASCII(dataTojson(options.data || "{}"))); 其中 options.data = 'page=3&limit=20'
	处理后，data= '{"limit":"20","page":"3"}'  是一个 json 字符串

- options.data = encrypt.encryptLong(data); 得到的是字符串长度 172 位，应该就是载荷中的参数

- sign = MD5(data + requestId + timestamp);  通过 md5 得到 sign 的值

由于上面使用的都是标准函数，从上述信息可以纯算加密结果
