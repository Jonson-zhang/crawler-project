## 二、SPA 架构概览

| 文件 | 大小 | 内容 |
|------|------|------|

## 三、网络请求分析

进入采购信息页面后，切换任一 tab，会依次发起两个请求：

### 3.1 公钥请求

```
POST https://ecuat.minmetals.com.cn/open/homepage/public
Headers: Content-Type: application/json
Body:    (空)
Response: MIGfMA0GCSqGSIb3DQEBAQUAA4GNADCBiQKBgQDvmuZXb4hH... (216 字符)
```

- 响应是 **Base64 编码的 DER 格式 RSA 公钥**（1024-bit，指数 65537）
- 每次请求返回的公钥相同（静态密钥）
- 需要携带 cookie（Jiasule CDN），但 cookie 内容不变

### 3.2 数据请求

```
POST https://ecuat.minmetals.com.cn/open/homepage/zbs/by-lx-page
Headers: Content-Type: application/json
Body:    {"param":"l5Zeu4wOjiYhjU9R6QbVsolXquQbRxZtNOobwRMgxSYn..."} (684 字符)
Response: {"currPage":1,"list":[...], ...}
```

- `param` 是加密后的 JSON 字符串，长度恒为 **684 字符**
- 响应是明文 JSON，直接包含数据列表

### 4.1 断点定位

### 4.2 加密调用链

拦截后返回主程序，核心代码如下：

```javascript
function P() {
    return P = h(g.a.mark((function A() {
        var e, t, n, r, a, s, i = arguments;
        return g.a.wrap((function(A) {

                    A.next = 4,

                case 4:

                    r = n.data,                            // r = 公钥(Base64)
                    t.setPublicKey(r),                     // ← 步骤② 导入公钥
                    a = m(m({}, e), {}, {
                        sign: B()(JSON.stringify(e)),      // ← 步骤③ 生成 MD5 签名
                        timeStamp: +new Date               // ← 步骤④ 加时间戳
                    }),
                    s = t.encryptLong(JSON.stringify(a)),  // ← 步骤⑤ RSA 加密
                    A.abrupt("return", s);
                case 10:
                case "end":
                    return A.stop()
                }
        }
        ), A)
    }
    ))),
    P.apply(this, arguments)

```

**调用链总结：**

```
① POST /open/homepage/public  →  获取 Base64 公钥 r
② t.setPublicKey(r)           →  导入 JSEncrypt
③ B()(JSON.stringify(e))     →  对原始参数 JSON 做 MD5 → sign
④ 合并参数 a = {...e, sign, timeStamp}
⑤ t.encryptLong(JSON.stringify(a)) → 加密后的 param (684 字符)
```

### 4.3 各环节详解

#### 参数 e（原始查询参数）

```json
{
    "inviteMethod": "",
    "businessClassfication": "",
    "mc": "",
    "lx": "ZBJG",       ← 不同 tab 对应不同值（见下方 LX 码表）
    "dwmc": "",
    "pageIndex": 1
}
```

对于 e，有些链接没有 `"businessClassfication"` 这一项，`"lx"` 的值也不同，最后就是 `"pageIndex"`。

#### B() — 签名函数

在控制台输入 `B()`，查看 B 函数，发现是一个 md5：

```javascript
function B() {
    return new Md5(!0).update(t)[e]()   // → 32 位小写十六进制 MD5
}
```

示例输出：`d93357811a689a88ee7c753e14059c86`

#### m() — 对象合并

等同于 `Object.assign`（展开运算符），将 sign 和 timeStamp 合并到 e 对象中。

```javascript
function m(A) {
    for (var e = 1; e < arguments.length; e++) {
        var t = null != arguments[e] ? arguments[e] : {};
        e % 2 ? v(Object(t), !0).forEach(function(e) { b(A, e, t[e]) })
              : Object.getOwnPropertyDescriptors 
                    ? Object.defineProperties(A, Object.getOwnPropertyDescriptors(t))
                    : v(Object(t)).forEach(function(e) {
                          Object.defineProperty(A, e, Object.getOwnPropertyDescriptor(t, e))
                      })
    }
    return A
}
```

其中又用到 b 函数和 v 函数，都在 m 函数的前后位置，就不一一列出。

**总结：**
- 先请求一次 public，得到响应值，给变量 r，用于生成公钥
- 通过上面的 e、B()、m()，可以得到参数 a
- 最后通过 `t.encryptLong(JSON.stringify(a))`，得到载荷参数

## 五、encryptLong 逆向（关键！）

> [!warning]
> **不要**假设 `encryptLong` 就是标准 JSEncrypt 的 `encryptLong`。该站点在 `index.js` 中**自定义覆写**了此方法，分块逻辑和编码方式都不标准。

### 5.1 获取 encryptLong 源码

在 hook 断点处执行 `.toString()` —— 这是拿到完整源码最快的方式：

```javascript
// 在断点暂停时，控制台执行
t.encryptLong.toString()
```

得到完整源码：

```javascript
d["a"].prototype.encryptLong = function (A) {
    var e = this.getKey(),
        t = ((e.n.bitLength() + 7) >> 3) - 11;   // = 117 (1024-bit key)
    try {
        var n = "", r = "";
        if (A.length > t)
            return (
                (n = A.match(/.{1,50}/g)),         // ← 关键！按 50 字符分块
                n.forEach(function (A) {
                    var t = e.encrypt(A);           // → RSA 加密 → hex
                    r += t;                         // → 拼接 hex
                }),
                w(r)                                // ← hex → base64
            );
        var a = e.encrypt(A), s = w(a);
        return s;
    } catch (i) {
        return i;
    }
};
```

### 5.2 关键发现：三个差异点

| 项目 | 标准 JSEncrypt | 该网站实际实现 |
|------|---------------|---------------|
| 分块大小 | **117 字节**（按 PKCS1 填充倒推） | **50 字符**（`A.match(/.{1,50}/g)`），按 UTF-8 编码后实际小于 117 字节 |
| 加密输出 | hex 字符串（不定长） | hex **补齐**到 2 × keySize = **256 字符** |
| 最终编码 | 纯 hex 拼接 | `w()` 函数做 **hex → base64** 转换 |

### 5.3 RSA 内部 encrypt 方法

```javascript
e.prototype.encrypt = function (e) {
    var t = (this.n.bitLength() + 7) >> 3,    // 128 (1024-bit key)
        n = se(e, t);                          // PKCS#1 v1.5 填充
    if (null == n) return null;
    var r = this.doPublic(n);                  // 模幂运算 m^e mod n
    if (null == r) return null;
    for (var o = r.toString(16), i = o.length, a = 0; a < 2 * t - i; a++)
        o = "0" + o;                            // 左侧补 0 到 256 字符
    return o;
};
```

### 5.4 hex2b64 编码函数 w()

同样用 `.toString()` 获取：

```javascript
function p(e) {
    var t, n, r = "";
    for (t = 0; t + 3 <= e.length; t += 3)
        ((n = parseInt(e.substring(t, t + 3), 16)),
         (r += f.charAt(n >> 6) + f.charAt(63 & n)));
    t + 1 == e.length
        ? ((n = parseInt(e.substring(t, t + 1), 16)), (r += f.charAt(n << 2)))
        : t + 2 == e.length &&
          ((n = parseInt(e.substring(t, t + 2), 16)),
           (r += f.charAt(n >> 2) + f.charAt((3 & n) << 4)));
    while ((3 & r.length) > 0) r += h;
    return r;
}
```

- `f` = `"ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789+/"`（标准 Base64 字母表）
- `h` = `"="`（Base64 填充符）

## 六、PKCS#1 v1.5 填充函数 se()

```javascript
function se(e, t) {
    if (t < e.length + 11) return (console.error("Message too long for RSA"), null);
    var n = [], r = e.length - 1;
    while (r >= 0 && t > 0) {
        var o = e.charCodeAt(r--);
        o < 128
            ? (n[--t] = o)
            : o > 127 && o < 2048
                ? ((n[--t] = (63 & o) | 128), (n[--t] = (o >> 6) | 192))
                : ((n[--t] = (63 & o) | 128), (n[--t] = ((o >> 6) & 63) | 128), (n[--t] = (o >> 12) | 224));
    }
    n[--t] = 0;
    var i = new ie(), a = [];
    while (t > 2) { a[0] = 0; while (0 == a[0]) i.nextBytes(a); n[--t] = a[0]; }
    return ((n[--t] = 2), (n[--t] = 0), new D(n));
}
```

格式：`00 02 [随机非零字节] 00 [原始数据]`
其中 `ie()` 是安全随机数生成器。

## 七、完整加密流程图

```
输入: {lx:"ZBGG", pageIndex:1, ...}
            │
            ▼
    JSON.stringify        → "{...lx:ZBGG...}" (无空格，冒号逗号分隔)
            │
      ┌─────┴─────┐
      │             │
      ▼             ▼
   MD5 hex       合并 sign +
   (B函数)       timeStamp
      │             │
      └──────┬──────┘
             ▼
   完整 JSON: {lx, sign, timeStamp}    (~158 字符)
             │
             ▼
    encryptLong (自定义覆写版):
    ┌─────────────────────────────────────┐
    │ 1. 按 50 字符分块 (/.{1,50}/g)        │
    │    chunk 0: 50 chars = 50 bytes      │
    │    chunk 1: 50 chars = 50 bytes      │
    │    chunk 2: 50 chars = 50 bytes      │
    │    chunk 3: 8 chars  = 8 bytes       │
    │                                     │
    │ 2. 每块 RSA 加密 (PKCS1v1.5)         │
    │    50 bytes + 11 padding ≤ 128      │
    │    输出: 128 bytes → hex zfill(256)  │
    │                                     │
    │ 3. hex 拼接: 256×4 = 1024 字符        │
    │                                     │
    │ 4. hex2b64: 1024 hex → 684 base64   │
    └─────────────────┬───────────────────┘
                      ▼
    POST /zbs/by-lx-page
    Body: {"param":"684字符..."}
```

## 十、使用 js-reverse MCP 逆向的步骤

以下是用 Claude Code 内置 MCP 工具链的实际操作记录，作为"工具化逆向"的标准流程参考：

### 10.2 逆向操作流程

| 步骤 | MCP 工具 | 操作 | 产出的信息 |
|------|----------|------|-----------|
| 1 | `navigate_page` | 打开 `https://ecuat.minmetals.com.cn/logonAction.do` | 页面加载，JS 注入 |
| 2 | `list_scripts` | 列出所有加载的脚本 | 3 个 JS 文件 URL |

| 4 | `search_in_sources` | 搜索 `"encryptLong"` | 定位调用位置和定义位置 |
| 5 | `search_in_sources` | 搜索 `"prototype.encryptLong"` | 定位覆写定义 |
| 6 | `get_script_source` / `Read` | 读取 `encryptLong` 函数附近的完整源码 | 分块大小 50、hex2b64 调用 |
| 7 | `Grep` + `Read` | 搜索 `parseInt.*substring.*16` 在 vendors.js 中定位 `p()` 函数 | hex2b64 完整实现 |

| 12 | `list_network_requests` | 查看 `reqid=115` 的完整 request/response | 确认 Headers、Cookie、响应结构 |

### 10.3 MCP vs 手动 curl+grep 对比

| 维度 | 手动 curl+grep | MCP 工具链 |
|------|---------------|-----------|
| 获取 JS 源码 | curl 下载压缩 JS → grep 盲搜 | `save_script_source` 自动格式化，直接可读 |
| 定位加密函数 | 从笔记推断调用链 → 猜测函数语义 | `search_in_sources` 精确搜索 → `get_script_source` 读上下文 |
| 验证加密结果 | 写 Python → 调 API → 看是否报"非法请求" | `break_on_xhr` 拦截真实请求 → 直接对比浏览器发出的 param 值 |
| 发现分块差异 | 报错后回头 grep 源码找 `.match` 正则 | 格式化源码中 `/.{1,50}/g` 一眼可见 |
| 发现 hex2b64 | 同上轮调试 | Grep `parseInt.*substring.*16` 即时定位 `p()` 函数 |

> [!tip] 核心优势
> 用 MCP 的最大优势是**零猜测**：`break_on_xhr` 拦截到浏览器发出的真实请求参数，可以直接作为"标准答案"。加密结果写对了就是对，不需要拿 Python 产物去服务器上试错。
